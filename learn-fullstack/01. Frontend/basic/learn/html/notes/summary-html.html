1 · Pendahuluan & Cara Kerja Web

HTML (HyperText Markup Language) adalah bahasa markup untuk membuat struktur halaman web. Bukan bahasa pemrograman — tidak memiliki logika, hanya mendefinisikan elemen dan makna konten.

  • Markup Language → sistem anotasi teks menggunakan tag yang memberikan makna dan instruksi kepada mesin. HTML untuk tampilan, XML untuk data, Markdown untuk teks sederhana.
  • Frontend Development → tiga pilar: HTML (struktur), CSS (presentasi), JavaScript (perilaku).
  • How the web works: Browser mengirim permintaan HTTP ke server → server merespon dengan file HTML → browser mem-parse HTML, membangun DOM → memuat CSS/JS → menampilkan halaman.
  • Komponen kunci: HTTP (protokol komunikasi), DNS (menerjemahkan domain ke IP), domain name, hosting (tempat file disimpan), browser (penafsir HTML).
  • SEO (Search Engine Optimization): praktik mengoptimasi halaman agar mudah ditemukan mesin pencari → struktur semantik, meta tags, kecepatan, aksesibilitas.
⚡ Prinsip kunci: HTML hanya mengatur struktur dan makna. Tampilan adalah tugas CSS, interaksi tugas JavaScript.
2 · Dasar‑dasar HTML

Struktur dokumen minimal:

<!DOCTYPE html>
<html lang="id">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Judul Halaman</title>
</head>
<body>
  <!-- konten halaman -->
</body>
</html>
  • <!DOCTYPE html> → deklarasi tipe dokumen, wajib di awal.
  • <html> → elemen akar, atribut lang menentukan bahasa konten.
  • <head> → metadata, tidak tampil di halaman.
  • <meta charset="UTF-8"> → encoding karakter, wajib.
  • <body> → semua konten yang terlihat.

Aturan dasar: Tag case‑insensitive (tapi rekomendasi lowercase). Komentar <!-- ... -->. Whitespace berlebih diabaikan browser. HTML Entities: &lt; untuk <, &gt; untuk >, &amp; untuk &, &nbsp; untuk spasi tak terputus.

3 · Textual Tags & Links
Headings & Paragraf
  • <h1> – <h6> → hierarki konten, <h1> hanya sekali per halaman.
  • <p> → paragraf naratif.
  • <hr> → garis horizontal (pergantian tema).
  • <br> → baris baru (alamat, puisi).
  • <pre> → teks preformatted (mempertahankan spasi).
Inline Text Semantics
  • <strong> → penting secara semantik (default bold).
  • <b> → bold visual saja (tanpa makna).
  • <em> → penekanan stres (default italic).
  • <i> → italic visual (istilah asing, nama ilmiah).
  • <mark> → teks ditandai (relevan konteks).
  • <sub> / <sup> → subscript / superscript.
  • <code> → kode program inline.
Links (Anchor)
  • <a href="URL"> → tautan ke halaman lain, anchor, email, telepon.
  • target="_blank" → buka di tab baru, selalu sertakan rel="noopener noreferrer" untuk keamanan.
  • Teks link harus deskriptif: "baca selengkapnya" bukan "klik di sini".
4 · Grouping Text & Standard Attributes
Grouping Containers
  • <div> → block-level container tanpa makna (untuk layout).
  • <span> → inline container tanpa makna (untuk styling).
  • Gunakan semantik bila ada, baru <div>/<span> jika tidak ada tag yang cocok.
Standard / Global Attributes
  • id → identifier unik (untuk CSS, JS, anchor).
  • class → pengelompokan elemen (styling, JS).
  • data-* → menyimpan data kustom.
  • style → inline CSS (hindari kecuali dinamis).
  • lang, title, tabindex, aria-* (aksesibilitas).
5 · Lists & Tables
Lists
  • <ul> → unordered list (bullet).
  • <ol> → ordered list (angka/huruf), atribut type, start, reversed.
  • <dl> + <dt> + <dd> → definition list (glosarium).
  • Nested list: letakkan <ul> di dalam <li>, bukan langsung di bawah list.
Table Tag
  • <table> → data tabular (bukan layout!).
  • <caption> → judul tabel (wajib untuk aksesibilitas).
  • <thead>, <tbody>, <tfoot> → struktur.
  • <th scope="col"> / scope="row" → header kolom/baris.
  • colspan / rowspan → menggabungkan sel.
6 · Embedding Media
Images
  • <img src="..." alt="..." width="..." height="..." loading="lazy" fetchpriority="high">.
  • Alt wajib: deskriptif untuk konten, kosong (alt="") untuk dekoratif.
  • <figure> + <figcaption> → gambar dengan keterangan yang terikat secara semantik.
  • loading="lazy" → muat saat mendekati viewport (performa).
  • fetchpriority="high" → prioritas tinggi (gambar hero).
Audio, Video, iframe
  • <audio controls><source src="..." type="..."></audio>.
  • <video controls poster="..." width height><source></video>.
  • <iframe title="..." src="..." loading="lazy" sandbox> → embed halaman lain (YouTube, map).
  • CSP (Content Security Policy): header atau meta tag untuk membatasi sumber yang boleh dimuat (pertahanan XSS).
7 · Using Forms

<form action="/proses" method="POST" enctype="multipart/form-data"> → mengirim data ke server.

Input & Labels
  • Setiap <input> harus punya <label for="id"> yang terhubung.
  • Tipe input: text, email, number, password, date, checkbox, radio, file, submit.
  • required, minlength, maxlength, min, max, pattern untuk validasi client‑side.
  • <textarea> → area teks multi‑baris.
  • <select> + <option> → dropdown.
File Upload & Limitations
  • <input type="file" accept="image/*" multiple> → unggah file.
  • Validasi HTML hanya untuk UX, keamanan tetap harus divalidasi di server.
  • Batasan: tidak ada validasi custom tanpa JavaScript, file upload membutuhkan enctype="multipart/form-data".
8 · Semantic Markup & Layout
Highlighting Changes & Quotations
  • <del datetime> → teks yang dihapus, <ins> → teks ditambahkan, <s> → tidak relevan lagi (harga lama).
  • <abbr title="..."> → singkatan dengan tooltip.
  • <cite> → judul karya (buku, film).
  • <dfn> → definisi pertama istilah.
  • <address> → informasi kontak penulis/toko.
  • <blockquote cite> → kutipan panjang, <q> → kutipan inline.
Layout Tags (HTML5)
  • <header> → pengantar, logo, navigasi.
  • <nav> → navigasi utama.
  • <main> → konten utama (hanya satu).
  • <section> → kelompok tematik (butuh heading).
  • <article> → konten independen (postingan, berita).
  • <aside> → konten pelengkap (sidebar).
  • <footer> → footer halaman/section.
✨ Gunakan heading (<h2> – <h6>) untuk setiap <section>/<article>, dan beri id agar bisa dirujuk dengan aria-labelledby.
9 · Styling Basics & Including JavaScript
  • Inline CSS: style="property: value;" → specificity tertinggi, hindari untuk maintainability.
  • Internal CSS: <style> ... </style> di <head>, cocok untuk halaman tunggal.
  • External CSS: <link rel="stylesheet" href="style.css"> → standar industri, reusable, caching.
  • JavaScript: <script src="script.js" defer></script> → defer menunda eksekusi hingga HTML selesai di‑parse, tidak memblokir rendering.
  • async → eksekusi segera setelah file diunduh (cocok untuk analitik).
10 · Accessibility & Basics of SEO
Accessibility (a11y)
  • WCAG POUR: Perceivable, Operable, Understandable, Robust.
  • Gunakan semantic HTML sebagai fondasi.
  • Focus indicator yang jelas (:focus-visible).
  • alt pada gambar, label pada form.
  • ARIA: aria-label, aria-labelledby, role hanya jika semantic tidak cukup.
  • skip link untuk lompat ke konten utama.
Basics of SEO
  • Crawling, indexing, ranking → mesin pencari membaca HTML.
  • <title> (50‑60 karakter), meta name="description" (155 karakter).
  • meta name="robots" content="index, follow".
  • link rel="canonical" → URL utama.
  • Open Graph tags: og:title, og:description, og:image, og:type untuk media sosial.
  • Structured data (JSON-LD): <script type="application/ld+json">{...}</script> untuk rich snippets.
  • GEO (Generative Engine Optimization): konten terstruktur dan faktual agar dikutip oleh AI search.

13 · Glosarium
DOM (Document Object Model)konsep
Representasi HTML sebagai pohon di memori browser, dimanipulasi oleh JavaScript.
Semantic HTMLkonsep
Memilih tag berdasarkan makna konten, bukan tampilan. Meningkatkan aksesibilitas dan SEO.
Self‑closing tagsintaks
Tag tanpa penutup, contoh <img>, <br>, <input>.

Block: ambil lebar penuh, baris baru. Inline: mengalir dalam teks, tidak bisa diatur lebar.

Structured Data / JSON‑LDSEO
Data terstruktur yang membantu mesin pencari memahami konten dan menampilkan rich snippets.
14 · Flash Cards
Apa perbedaan <strong> dan <b>?
<strong> memiliki makna semantik (penting), <b> hanya visual bold. Screen reader membaca <strong> dengan penekanan.
Mengapa menggunakan <figure> untuk gambar?
<figure> + <figcaption> mengelompokkan gambar dengan keterangan yang secara semantik terikat, berbeda dengan <img> + teks di luar.
Apa itu defer pada <script>?
Menunda eksekusi script hingga HTML selesai di‑parse, mencegah render‑blocking, urutan eksekusi sesuai deklarasi.
Kapan menggunakan <section> vs <article>?
<section> untuk kelompok tematik yang terkait dengan halaman; <article> untuk konten independen yang bisa berdiri sendiri (postingan, berita).
Apa fungsi atribut scope pada tabel?
scope="col" pada <th> menandakan header kolom, scope="row" header baris, membantu screen reader memahami struktur tabel.
15 · Best Practices & Anti‑Patterns
✅ Pola yang Selalu Digunakan
  • Gunakan semantic HTML: <nav>, <main>, <section>, bukan <div> semua.
  • Setiap <input> harus punya <label for> yang terhubung.
  • Link eksternal: target="_blank" rel="noopener noreferrer".
  • Alt pada gambar: deskriptif untuk konten, kosong (alt="") untuk dekoratif.
  • Gunakan width dan height pada <img> untuk mencegah layout shift (CLS).
  • Sertakan lang="id" pada <html>.
❌ Anti‑Pattern yang Harus Dihindari
  • Menggunakan <table> untuk layout.
  • <br><br> untuk spasi vertikal → gunakan CSS margin.
  • <b> dan <i> untuk makna penting (gunakan <strong>/<em>).
  • width="300px" pada <img> → atribut hanya menerima angka tanpa satuan.
  • border="1" pada tabel → styling dengan CSS.
  • innerHTML untuk menampilkan input pengguna (rentan XSS), gunakan textContent.
