03 · CSS Basics: Selectors, Cascade, Specificity

Selectors: Cara Menarget Elemen

/* SIMPLE SELECTORS */
*          { box-sizing: border-box; }   /* universal — semua elemen */
p          { line-height: 1.7; }         /* element — semua tag p */
.kartu     { background: white; }        /* class — elemen dengan class ini */
#kasir     { background: #f9f6f0; }      /* ID — satu elemen unik */
h1, h2, h3 { color: #1b5e20; }          /* grouping — berbagi aturan */

/* COMBINATOR SELECTORS */
.kartu p        { color: #444; }         /* descendant — p di mana saja di dalam .kartu */
.kartu > h2     { font-size: 1.5rem; }  /* child — h2 langsung di dalam .kartu */
h2 + p          { font-weight: bold; }   /* next sibling — p langsung setelah h2 */
h2 ~ p          { color: #555; }         /* subsequent sibling — semua p setelah h2 */

/* ATTRIBUTE SELECTORS */
input[type="email"]       { border-color: blue; }
input[aria-required="true"] { border-left: 3px solid red; }
a[href^="https"]          { color: green; }  /* href DIMULAI dengan https */
a[href$=".pdf"]           { color: orange; } /* href DIAKHIRI dengan .pdf */

/* PSEUDO-CLASSES — state elemen */
a:hover          { color: darkgreen; }
input:focus      { outline: 2px solid blue; }
input:invalid    { border-color: red; }
li:first-child   { border-top: none; }
li:nth-child(2n) { background: #f5f5f5; }

/* PSEUDO-ELEMENTS — konten virtual */
h2::before { content: ""; display: block; width: 40px; height: 3px; background: green; }
p::first-line { font-weight: bold; }
.btn::after { content: " →"; }

Cascade: Urutan Prioritas

Ketika dua aturan berkonflik, browser menggunakan urutan ini dari terendah ke tertinggi:

Level Sumber Contoh
1 (terendah) Browser default h1 { font-size: 2em }
2 User stylesheet Preferensi aksesibilitas user
3 Author external CSS main.css
4 Author internal CSS <style> di head
5 Author inline style style="..."
6 (tertinggi) !important color: red !important

Di dalam setiap level, specificity menentukan pemenang. Jika specificity sama, yang ditulis terakhir dalam source order menang.

Specificity: Sistem Skor (ID, Class, Tag)

/* Format: (ID, Class/Attr/PseudoClass, Tag) */

p                              /* (0, 0, 1) = 1 */
.harga                         /* (0, 1, 0) = 10 */
p.harga                        /* (0, 1, 1) = 11 */
.kartu .harga                  /* (0, 2, 0) = 20 */
.kartu .kartu__harga           /* (0, 2, 0) = 20 */
#kasir .harga                  /* (1, 1, 0) = 110 */  ← menang atas semua di atas

/* Satu ID (1,0,0) SELALU mengalahkan berapapun class.
   Tidak ada "carry over" — ini bukan desimal biasa. */

/* TIPS PRODUKSI: hindari ID selector di CSS.
   Gunakan class saja agar specificity tetap rendah dan mudah di-override. */
⚠️ Jika kamu sering menulis !important, itu sinyal bahwa struktur selector CSS-mu perlu direstrukturisasi — kamu sedang "berkelahi" dengan cascade, bukan bekerja bersamanya.

Inheritance: Warisan Otomatis

/* Properti yang DIWARISKAN (properti teks) */
body { font-family: Georgia, serif; color: #333; line-height: 1.7; }
/* ↑ Semua child mewarisi ini secara otomatis */

/* Properti yang TIDAK diwariskan (properti kotak/layout) */
/* margin, padding, border, background, width — tidak diwariskan */

/* Paksa mewarisi dengan kata kunci 'inherit' */
button, input, select, textarea {
  font-family: inherit; /* ← tanpa ini, browser pakai font default sendiri */
}

/* Reset ke nilai default browser dengan 'initial' */
.reset { color: initial; }
