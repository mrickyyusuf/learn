2 · Cheatsheet — Quick Reference

Selectors Cheatsheet

Simple Selectors
  • * — universal (semua elemen)
  • p — element/tag selector
  • .nama — class selector
  • #id — ID (hindari di CSS!)
  • h1, h2, h3 — grouping
  • [attr="val"] — attribute
Combinators
  • A B — descendant (B di mana saja di dalam A)
  • A > B — child langsung A
  • A + B — sibling langsung setelah A
  • A ~ B — semua sibling setelah A
Pseudo Classes
  • :hover :focus :active
  • :focus-visible — keyboard only
  • :first-child :last-child
  • :nth-child(n) :nth-child(2n)
  • :not(.cls) — negasi
  • :has(child) — parent selector
  • :valid :invalid :checked
Pseudo Elements
  • ::before — konten sebelum (wajib content:)
  • ::after — konten setelah
  • ::first-line — baris pertama
  • ::first-letter — huruf pertama
  • ::placeholder — placeholder input
  • ::selection — teks yang diseleksi

Box Model

box-sizing: border-box (SELALU gunakan ini)
content lebar konten asli
padding jarak konten ke border, termasuk dalam width
border garis tepi, termasuk dalam width
margin jarak ke elemen lain, di luar width
⚠️ outline tidak termasuk dalam box model — tidak mempengaruhi layout

Flexbox Quick Reference

/* CONTAINER */
display: flex;
flex-direction: row | column | row-reverse | column-reverse;
justify-content: flex-start | center | flex-end | space-between | space-around | space-evenly;
align-items: stretch | flex-start | center | flex-end | baseline;
flex-wrap: nowrap | wrap;
gap: 16px;

/* ITEM */
flex: 1;               /* grow shrink basis shorthand */
flex: 0 0 200px;       /* lebar tetap */
align-self: center;    /* override align-items per item */
order: -1;             /* ubah urutan visual */

Grid Quick Reference

/* CONTAINER */
display: grid;
grid-template-columns: repeat(3, 1fr);
grid-template-columns: repeat(auto-fill, minmax(200px, 1fr)); /* responsif */
gap: 16px;
grid-template-areas:
  "header header"
  "sidebar main";

/* ITEM */
grid-column: span 2;   /* mencakup 2 kolom */
grid-row: span 2;      /* mencakup 2 baris */
grid-area: header;     /* named area */

Animation Quick Reference

/* TRANSITION */
transition: transform 0.2s ease, opacity 0.2s ease;
/* ✅ Gunakan HANYA transform dan opacity untuk animasi GPU */

/* TRANSFORM */
transform: translateY(-4px) scale(1.05);
transform: translate(-50%, -50%); /* centering trick */

/* KEYFRAME */
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}
.elemen { animation: fadeIn 0.5s ease forwards; }
3 · Cascade, Specificity & Inheritance

Urutan Prioritas Cascade

Dari terendah ke tertinggi: browser default → user stylesheet → author external → author internal → author inline → !important.

Specificity Scoring

/* Format: (ID, Class/Attr/Pseudo, Tag) — kolom kiri SELALU mengalahkan kolom kanan */
p                    /* (0,0,1) */
.harga               /* (0,1,0) */
p.harga              /* (0,1,1) */
.kartu .harga        /* (0,2,0) */
#kasir .harga        /* (1,1,0) — satu ID mengalahkan berapapun class */

Properti yang Diwariskan vs Tidak

Properti teks (color, font-family, line-height) diwariskan otomatis. Properti kotak (margin, padding, border, background) tidak diwariskan.

4 · Layout Systems
Pilih Layout yang Tepat
  • Teks mengalir → flow layout
  • Satu baris/kolom → Flexbox
  • Baris DAN kolom → Grid
  • Badge, tooltip → position: absolute
  • Navbar sticky → position: fixed/sticky
Position Values
  • static — default, tidak ada efek top/left
  • relative — offset dari posisi normal
  • absolute — relatif ke nearest positioned ancestor
  • fixed — relatif ke viewport
  • sticky — normal flow + menempel saat scroll
5 · Studi Kasus: Toko Sembako

Design Tokens (CSS Variables)

:root {
  --warna-brand: #2e7d32;         /* hijau toko */
  --warna-brand-gelap: #1b5e20;
  --warna-brand-muda: #e8f5e9;
  --warna-bahaya: #c62828;        /* stok kritis */
  --warna-bg: #f4f7fc;
  --warna-bg-kartu: #ffffff;
  --shadow-sm: 0 2px 8px rgba(0,0,0,0.05);
  --shadow-md: 0 4px 16px rgba(0,0,0,0.08);
  --radius-md: 8px;
  --transisi: 0.2s ease;
}

Kartu Produk — BEM + CSS Variables

/* Block */
.kartu-produk {
  position: relative;                        /* container untuk badge ::after */
  background: var(--warna-bg-kartu);
  border: 1px solid #e0e0e0;
  border-radius: var(--radius-md);
  padding: 1rem;
  transition: transform var(--transisi), box-shadow var(--transisi);
}

.kartu-produk:hover {
  transform: translateY(-3px);
  box-shadow: var(--shadow-md);
}

/* Elements */
.kartu-produk__judul { font-size: 1rem; font-weight: bold; margin-bottom: .25rem; }
.kartu-produk__harga { color: var(--warna-brand); font-size: 1.2rem; font-weight: bold; }
.kartu-produk__tombol { background: var(--warna-brand); color: white; border: none; padding: .5rem 1rem; border-radius: 6px; cursor: pointer; }

/* Modifiers */
.kartu-produk--stok-kritis { border-left: 4px solid var(--warna-bahaya); }
.kartu-produk--stok-kritis::after { content: "⚠ Stok Kritis"; position: absolute; top: 8px; right: 8px; background: var(--warna-bahaya); color: white; font-size: .7rem; padding: 2px 8px; border-radius: 4px; }

/* Grid responsif tanpa media query */
.grid-produk { display: grid; grid-template-columns: repeat(auto-fill, minmax(200px, 1fr)); gap: 1rem; }
6 · Glosarium
Cascade konsep
Sistem prioritas CSS yang menentukan aturan mana yang berlaku saat terjadi konflik. Urutan: browser default → user → author external → internal → inline → !important.
Specificity konsep
Sistem skor CSS (ID, Class, Tag) yang menentukan pemenang ketika dua aturan berkonflik dalam level cascade yang sama. Satu ID selalu mengalahkan berapapun class.
Box Model layout
Model yang merepresentasikan setiap elemen HTML sebagai kotak berlapis: content → padding → border → margin. Dengan box-sizing: border-box, width mencakup padding dan border.
Flexbox layout
Layout system satu dimensi (baris atau kolom) yang memudahkan alignment dan distribusi ruang antar item. Diaktifkan dengan display: flex pada container.
CSS Grid layout
Layout system dua dimensi (baris dan kolom secara bersamaan). Ideal untuk layout halaman dan grid produk. Pola auto-fill minmax membuat grid responsif tanpa media query.
Stacking Context layout
Konteks tiga dimensi dalam halaman yang dibuat oleh elemen dengan z-index dan position, opacity < 1, atau transform. Z-index hanya bermakna di dalam stacking context yang sama.
Custom Properties / CSS Variables variabel
Variabel CSS yang didefinisikan dengan -- prefix, diakses dengan var(). Fondasi design system dan theming. Bisa dimanipulasi JavaScript dan mendukung dark mode.
BEM metodologi
Block Element Modifier — konvensi penamaan class CSS. .block__element--modifier. Menjaga specificity (0,1,0) secara konsisten dan mencegah konflik nama.
Critical CSS performa
Subset CSS minimal yang diperlukan untuk merender konten above-the-fold. Di-inline di head untuk mempercepat First Contentful Paint.
Repaint vs Reflow performa
Reflow (Layout) terjadi ketika dimensi/posisi berubah — mahal. Repaint terjadi ketika warna/visual berubah — lebih murah. Transform dan opacity hanya memicu Composite — paling murah.
Media Query responsif
Kondisi CSS yang berlaku berdasarkan karakteristik viewport (lebar, orientasi, preferensi pengguna). Mobile-first menggunakan min-width.
Container Query responsif
Seperti media query tapi berdasarkan ukuran container elemen, bukan viewport. Membuat komponen benar-benar reusable di berbagai konteks layout.
7 · Flash Cards
Apa perbedaan display: none vs visibility: hidden?
display:none menghapus elemen dari layout (ruang tidak ada). visibility:hidden menyembunyikan visual tapi ruang tetap ada. Keduanya tidak terlihat pengguna, tapi display:none juga disembunyikan dari screen reader.
Kapan pakai position: absolute vs fixed?
absolute: relatif ke nearest positioned ancestor — untuk badge, tooltip, dropdown. fixed: relatif ke viewport — untuk navbar, chat button. Keduanya keluar dari normal flow.
Apa syarat pseudo-element ::before bekerja?
Wajib ada properti content: (boleh kosong ""). Tanpa content, elemen virtual tidak muncul di halaman.
Kenapa transform lebih smooth dari menganimasikan width?
transform hanya memicu Composite layer yang dieksekusi GPU. Menganimasikan width memicu Layout ulang di setiap frame — CPU yang mengerjakan, lebih lambat, menyebabkan jank pada 60fps.
Apa itu mobile-first approach?
Tulis style default untuk mobile, tambahkan media query min-width untuk layar lebih besar. Direkomendasikan karena mayoritas traffic e-commerce dari mobile, dan memaksa prioritas konten.
Bagaimana grid responsif tanpa media query?
grid-template-columns: repeat(auto-fill, minmax(200px, 1fr)). auto-fill buat kolom sebanyak muat. minmax(200px,1fr) tiap kolom min 200px max 1fr. Hasilnya otomatis responsif.
Apa perbedaan CSS Variable dan Sass Variable?
CSS Variable (--nama) hidup di browser, bisa diubah runtime via JS, mendukung dark mode. Sass Variable ($nama) dikompilasi menjadi nilai statis sebelum ke browser — tidak bisa diubah runtime.
Apa itu BEM dan mengapa digunakan?
Block__Element--Modifier. Konvensi naming class untuk mencegah konflik dan menjaga specificity tetap (0,1,0). Block=komponen, Element=bagian komponen (--), Modifier=variasi (__).
Apa fungsi clamp(min, preferred, max)?
Nilai yang dibatasi antara minimum dan maximum. Digunakan untuk responsive typography tanpa media query: clamp(1rem, 3vw, 2rem) — font yang mengikuti lebar layar tapi dibatasi ukurannya.
Kenapa !important harus dihindari?
!important membalikkan urutan cascade dan membuat kode sulit di-override. Jika sering dipakai, itu sinyal bahwa struktur selector CSS perlu direstrukturisasi. Gunakan hanya untuk library overrides atau utility yang memang harus menang.
8 · Tips & Anti-Pattern

Anti-Pattern yang Harus Dihindari

⚠️ Menggunakan !important sebagai solusi konflik — perbaiki struktur selector, bukan paksa dengan !important.
⚠️ ID selector di CSS (#kasir) — gunakan class, specificity ID terlalu tinggi dan susah di-override.
⚠️ Inline style untuk keputusan desain statis — gunakan class. Inline style hanya untuk nilai runtime dinamis.
⚠️ Menganimasikan width/height/top/left — gunakan transform. Animasi layout properties menyebabkan jank.
⚠️ outline: none tanpa pengganti — selalu ganti dengan focus-visible yang estetik.
⚠️ Mengandalkan display: none untuk aksesibilitas — gunakan visually-hidden class agar screen reader tetap bisa membaca.

Tips Produksi

✅ Selalu gunakan box-sizing: border-box pada *, *::before, *::after.
✅ Definisikan design tokens di :root — warna, spacing, shadow, radius.
✅ Gunakan :focus-visible bukan :focus untuk focus ring yang tidak mengganggu mouse user.
✅ Gunakan gap pada Flexbox/Grid daripada margin pada item individual.
✅ Pertimbangkan prefers-reduced-motion untuk pengguna yang sensitif terhadap animasi.
✅ Gunakan min-height: 44px pada touch target untuk aksesibilitas mobile.
✅ Naming class yang mendeskripsikan fungsi, bukan tampilan: .teks-penting bukan .teks-merah.
9 · Metodologi: Kapan Pakai Apa
Metodologi Keunggulan Digunakan di Cocok Untuk
BEM Naming konsisten, specificity terjaga HTML/CSS biasa, Sass Semua project
Sass/SCSS Nesting, mixin, partials Semua framework Project medium-large
PostCSS Autoprefixer, minification, modern CSS Build pipeline Semua project dengan build tool
