10 · Position Property

Konsep: Normal Flow vs Positioned Elements

Secara default, semua elemen mengikuti normal flow — block elements mengalir dari atas ke bawah, inline elements mengalir kiri ke kanan dalam baris. Properti position memungkinkan kamu "keluar" dari flow normal untuk menempatkan elemen di posisi yang lebih presisi. Tapi keluar dari flow ada konsekuensinya — elemen lain mungkin tidak "tahu" elemen itu ada.

Setiap nilai position (kecuali static) mengaktifkan properti top, right, bottom, left untuk mengatur posisi, serta membuat elemen menjadi positioned element yang relevan untuk z-index.

Perbandingan Cepat Lima Nilai

Value Referensi Posisi Pengaruh ke Layout Kapan Pakai
static Normal flow (default) Normal Default — tidak perlu ditulis eksplisit
relative Posisi normalnya sendiri Ruangnya tetap ada Offset kecil, container untuk absolute
absolute Nearest positioned ancestor Dikeluarkan dari flow Badge, tooltip, dropdown, modal
fixed Viewport (jendela browser) Dikeluarkan dari flow Navbar sticky, chat button, cookie banner
sticky Normal flow + viewport saat scroll Normal sampai threshold Header tabel sticky, section header

position: relative

Elemen tetap di flow normal (ruangnya dipertahankan), tapi secara visual digeser relatif terhadap posisi normalnya sendiri. Yang paling penting: relative membuat elemen menjadi containing block untuk child yang absolute.

Normal Relative: top:20 left:30 Absolute: top:10 right:10 
.kartu-produk {
  position: relative; /* ← menjadi "penjangkar" untuk badge di dalamnya */
}

.badge-diskon {
  position: absolute;
  top: 8px;
  right: 8px; /* ← dihitung dari .kartu-produk, bukan viewport */
}

position: absolute

Elemen dikeluarkan dari normal flow (elemen lain seolah ia tidak ada) dan diposisikan relatif terhadap nearest ancestor yang memiliki position selain static. Jika tidak ada, ia menggunakan viewport. Ini adalah pola yang hampir selalu dipakai bersama relative pada parent-nya.

⚠️ Kesalahan paling umum: lupa menambahkan position: relative pada parent, sehingga elemen absolute "kabur" ke posisi yang tidak diharapkan karena menggunakan viewport sebagai referensi. 
/* Pola klasik: parent relative, child absolute */
.container { position: relative; }
.overlay   { position: absolute; inset: 0; background: rgba(0,0,0,.5); }
/* inset: shorthand untuk top: 0; right: 0; bottom: 0; left: 0 */

position: fixed

Elemen menempel di posisi yang sama di viewport bahkan saat halaman di-scroll. Berguna untuk navbar yang selalu terlihat, tombol "kembali ke atas", atau chat widget. Karena keluar dari flow, perlu hati-hati agar tidak menutupi konten penting — beri padding pada elemen di bawahnya. 

.navbar-sticky {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;         /* atau width: 100% */
  z-index: 100;     /* pastikan di atas konten lain */
}

/* Kompensasi agar konten tidak tersembunyi di balik navbar */
body { padding-top: 56px; } /* tinggi navbar */

position: sticky

Berperilaku seperti relative sampai elemen mencapai threshold yang ditentukan saat scroll, lalu "menempel" seperti fixed sampai parent-nya keluar dari viewport. Sangat berguna untuk header tabel yang panjang atau section header dalam halaman artikel panjang.

Header Kategori: Bahan Pokok (sticky saat scroll)

Beras Premium 5kg — Rp 78.000

Gula Pasir 1kg — Rp 14.000

Minyak Goreng 2L — Rp 32.500

Tepung Terigu 1kg — Rp 12.000

Garam Halus — Rp 5.000

thead th {
  position: sticky;
  top: 0; /* menempel di posisi 0 dari atas container scroll */
  background: white; /* harus solid agar tidak transparan saat scroll */
  z-index: 1;
}
💡 sticky membutuhkan parent yang memiliki overflow yang sesuai dan tinggi yang terdefinisi. Jika tidak bekerja, periksa apakah ada ancestor dengan overflow: hidden yang memblokir scroll context.
