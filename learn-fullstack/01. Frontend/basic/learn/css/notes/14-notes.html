Normal Flow: Cara Default Browser Menyusun Elemen

Flow layout adalah sistem layout bawaan browser sebelum kamu menambahkan apapun. Dalam flow layout, setiap elemen mengikuti dua aturan sederhana: elemen block mengalir dari atas ke bawah (membentuk "block formatting context"), dan elemen inline mengalir dari kiri ke kanan dalam satu baris sebelum wrap ke baris berikutnya ("inline formatting context").

Memahami flow layout adalah fondasi penting sebelum belajar Flexbox dan Grid — karena Flexbox dan Grid pada dasarnya adalah cara untuk mengambil alih kendali dari flow layout dan mengganti dengan model yang lebih fleksibel.

Margin Collapse

Salah satu perilaku flow layout yang paling sering mengejutkan pemula adalah margin collapse. Ketika dua margin vertikal bertemu (margin-bottom elemen di atas bertemu margin-top elemen di bawah), keduanya tidak dijumlahkan — melainkan yang lebih besar yang berlaku. Ini hanya terjadi pada margin vertikal dalam flow normal, tidak pada Flexbox atau Grid. 

/* Contoh: dua p berturut-turut */
p { margin-bottom: 20px; margin-top: 20px; }
/* Jarak antar dua p = 20px (bukan 40px) karena margin collapse */

/* - elemen dengan overflow selain visible */
/* - elemen dengan border atau padding di antaranya */

Float — Warisan yang Masih Perlu Diketahui

Float awalnya dirancang untuk "gambar yang dikelilingi teks" seperti di majalah, bukan untuk layout halaman. Dulu dipakai untuk membuat layout kolom sebelum Flexbox ada. Sekarang hampir tidak dipakai untuk layout, tapi masih relevan untuk use case aslinya.

Gambar yang di-float

Teks ini mengalir mengelilingi elemen yang di-float kiri. Ini adalah use case asli float — mengelilingi gambar dengan teks seperti di majalah atau koran. Parent harus punya overflow: hidden atau clearfix agar container tidak collapse. 

img { float: left; margin: 0 16px 8px 0; }

/* Clearfix — cara klasik mengatasi container yang collapse karena float */
.clearfix::after {
  content: "";
  display: table;
  clear: both;
}

/* Cara modern: gunakan display: flow-root pada parent */
.container { display: flow-root; } /* membuat Block Formatting Context baru */

Multi-Column Layout

Multi-column layout berguna untuk konten teks panjang yang perlu dibagi ke beberapa kolom seperti koran atau majalah. Konten mengalir otomatis dari kolom ke kolom.

Toko sembako adalah tulang punggung ekonomi rakyat. Setiap hari jutaan keluarga bergantung pada ketersediaan bahan pokok yang terjangkau dan berkualitas. Dengan sistem manajemen yang baik, toko sembako bisa melayani lebih banyak pelanggan dengan lebih efisien.

Teknologi modern memungkinkan pengelolaan stok, kasir, dan pelaporan dilakukan secara digital — mengurangi kesalahan manusia dan meningkatkan kepuasan pelanggan secara signifikan. 

.artikel {
  column-count: 3;      /* 3 kolom */
  /* atau gunakan column-width untuk responsif: */
  column-width: 250px;  /* lebar minimum per kolom, otomatis hitung jumlahnya */
  column-gap: 2rem;
  column-rule: 1px solid #ddd; /* garis pemisah antar kolom */
}

/* Elemen yang tidak boleh terpotong antar kolom */
h2 { break-after: avoid; } /* h2 tidak boleh diikuti break kolom */
figure { break-inside: avoid; } /* gambar tidak boleh terpotong */
💡 Untuk layout halaman modern (kartu produk, grid konten, sidebar + main), gunakan Flexbox atau Grid (materi 15). Float dan multi-column hanya untuk use case teks/konten editorial.
