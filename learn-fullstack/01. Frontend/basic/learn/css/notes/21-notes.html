Mengapa Metodologi Diperlukan?

CSS yang ditulis tanpa struktur sistematis akan menjadi "spaghetti CSS" — tidak ada yang tahu apa yang aman dihapus, setiap penambahan rule takut merusak yang lain, dan selector !important mulai bermunculan di mana-mana. Metodologi adalah konvensi penamaan dan organisasi yang mencegah kekacauan ini terjadi ketika tim dan codebase bertumbuh.

BEM — Block, Element, Modifier

BEM adalah sistem penamaan class yang memisahkan komponen UI menjadi tiga bagian: Block (komponen independen), Element (bagian dari block), dan Modifier (variasi dari block atau element). Sintaksnya: .block__element--modifier.

Block: Komponen Independen

/* Block: bisa berdiri sendiri, bermakna tanpa konteks */
.kartu-produk { }
.navbar { }
.form-kasir { }

Element: Bagian dari Block (double underscore)

/* Element: bagian dari block, tidak bermakna di luar block-nya */
.kartu-produk__gambar { }
.kartu-produk__judul  { }
.kartu-produk__harga  { }
.kartu-produk__tombol { }

/* HTML */
<article class="kartu-produk">
  <img class="kartu-produk__gambar" ...>
  <h3 class="kartu-produk__judul">Beras Premium</h3>
  <p class="kartu-produk__harga">Rp 78.000</p>
  <button class="kartu-produk__tombol">Tambah</button>
</article>

Modifier: Variasi (double dash)

/* Modifier: variasi state atau tampilan dari block atau element */
.kartu-produk--stok-kritis { border-color: red; }
.kartu-produk--diskon      { }
.kartu-produk__harga--promo { color: red; }
.kartu-produk__tombol--disabled { opacity: 0.5; }

/* HTML: tambahkan modifier BERSAMA class dasar, bukan mengganti */
<article class="kartu-produk kartu-produk--stok-kritis">
  <p class="kartu-produk__harga kartu-produk__harga--promo">Rp 65.000</p>
</article>
💡 Keunggulan BEM: specificity SELALU (0,1,0) karena hanya class selector, tidak ada nesting, tidak ada konflik. Mudah di-maintain karena nama class sudah menjelaskan di mana ia berada dan apa hubungannya.

Sass / SCSS — CSS dengan Superpowers

Sass adalah preprocessor CSS — kamu menulis SCSS (Sass dengan sintaks mirip CSS), dan Sass mengkompilasinya menjadi CSS biasa. SCSS menambahkan fitur seperti variabel (sebelum CSS variables ada), nesting, mixin, dan partial yang membuat CSS jauh lebih DRY. 

// SCSS — file berekstensi .scss
// Variabel Sass (berbeda dari CSS variables — dikompilasi menjadi nilai statis)
$warna-brand: #2e7d32;
$spacing: 16px;

// Nesting — mencerminkan struktur HTML
.kartu-produk {
  padding: $spacing;
  border-radius: 8px;

  // Nested: dikompilasi menjadi .kartu-produk:hover
  &:hover { transform: translateY(-2px); }

  // Nested: dikompilasi menjadi .kartu-produk__judul
  &__judul { font-size: 1.1rem; color: $warna-brand; }

  // Modifier: dikompilasi menjadi .kartu-produk--diskon
  &--diskon { border-left: 4px solid $warna-brand; }
}

// Mixin — blok CSS yang bisa dipakai ulang
@mixin flex-center {
  display: flex;
  align-items: center;
  justify-content: center;
}

.hero { @include flex-center; height: 100vh; }

// Partial — file SCSS yang dimulai dengan _ tidak dikompilasi sendiri
// _variables.scss, _mixins.scss, _components.scss
// Diimport di main.scss:
@use 'variables';
@use 'mixins';
@use 'components/kartu-produk';

PostCSS — CSS melalui Plugin

PostCSS bukan bahasa baru — ia adalah tool yang memproses CSS menggunakan plugin JavaScript. Berbeda dari Sass yang punya fitur built-in, PostCSS sangat modular: kamu pilih hanya plugin yang kamu butuhkan. Plugin paling populer adalah Autoprefixer (otomatis tambahkan vendor prefix seperti -webkit-), cssnano (minifikasi CSS), dan postcss-preset-env (gunakan fitur CSS modern yang dikompilasi untuk browser lama). 

/* Kamu menulis CSS modern yang bersih */
.kartu { transition: transform 0.2s ease; }

/* Autoprefixer otomatis menambahkan prefix jika diperlukan */
.kartu {
  -webkit-transition: transform 0.2s ease;
  transition: transform 0.2s ease;
}

/* Di project Vite/webpack, PostCSS dijalankan otomatis sebagai bagian build */

CSS Modules — Scoped CSS

CSS Modules adalah teknik di mana setiap file CSS di-scope ke komponen yang menggunakannya — class names di-transform menjadi hash yang unik sehingga tidak ada konflik antar komponen. Sangat populer di ekosistem React. 

/* KartuProduk.module.css */
.kartu { padding: 16px; border-radius: 8px; }
.harga { color: #2e7d32; font-weight: bold; }

/* KartuProduk.jsx */
import styles from './KartuProduk.module.css';

function KartuProduk() {
  return (
    <article className={styles.kartu}>
      <p className={styles.harga}>Rp 78.000</p>
    </article>
  );
}

/* Di browser, class name menjadi sesuatu seperti: */
/* .KartuProduk_kartu__xK2Lp { padding: 16px; } */
/* Sehingga tidak bisa konflik dengan .kartu dari komponen lain */

CSS-in-JS — styled-components, Emotion

CSS-in-JS adalah pendekatan menulis CSS di dalam JavaScript. Styled-components adalah yang paling populer. Keunggulannya adalah colocating style dengan komponen, props-based styling yang powerful, dan automatic critical CSS extraction. 

// styled-components (React)
import styled from 'styled-components';

const KartuProduk = styled.article`
  padding: 16px;
  border-radius: 8px;
  border-left: 4px solid ${props => props.kritis ? 'red' : 'green'};
  background: ${props => props.kritis ? '#fce4ec' : 'white'};
  transition: transform 0.2s ease;

  &:hover { transform: translateY(-2px); }
`;

// Penggunaan — props langsung menentukan style
<KartuProduk kritis={stok < 5}>
  ...
</KartuProduk>

Perbandingan: Kapan Pakai Apa?

Metodologi Best For Ekosistem
BEM HTML/CSS murni, tim yang butuh konvensi naming Semua
SCSS/Sass Project medium-large, team konvensional Semua
PostCSS Build pipeline, autoprefixing, minification Semua
CSS Modules React/Vue, hindari naming conflict React, Vue
CSS-in-JS Dynamic styling, design systems di React React
Tailwind CSS Rapid prototyping, utility-first, design system Semua
💡 Untuk proyek retail kamu: mulai dengan BEM + SCSS untuk struktur yang solid. Saat masuk React nanti, gunakan CSS Modules atau Tailwind. Untuk design system yang scalable seperti Shopee, pertimbangkan kombinasi CSS Variables + Tailwind + CSS Modules.
