02 · Frontend Development: HTML, CSS, JS

Tiga Lapisan Frontend

Frontend development dibangun dari tiga teknologi yang bekerja bersama secara komplementer. Analoginya seperti sebuah toko fisik: HTML adalah bangunan fisiknya (dinding, rak, kasir), CSS adalah desain interiornya (warna, pencahayaan, tata letak), dan JavaScript adalah sistem operasionalnya (pintu otomatis, mesin kasir, alarm stok).

HTML dari Perspektif CSS

Ketika CSS engineer membaca HTML, yang dilihatnya bukan sekadar konten — melainkan kumpulan target yang bisa diseleksi. Setiap class, id, hubungan parent-child, dan state interaktif adalah titik yang bisa "diarahkan" oleh selector CSS.

HTML yang baik untuk CSS memberikan hooks (pengait) yang bermakna — bukan terlalu sedikit (CSS jadi penuh selector panjang rapuh) dan bukan terlalu banyak (HTML berantakan dengan class tidak bermakna). 

/* ❌ HTML tanpa CSS hooks */
<div><div>Beras 5kg</div><div>Rp 78.000</div></div>

/* ✅ HTML dengan hooks semantik */
<article class="kartu-produk" data-stok="42">
  <h3 class="kartu-produk__judul">Beras Premium 5kg</h3>
  <p class="kartu-produk__harga">Rp 78.000</p>
</article>

CSS: State-Driven Design

CSS mendefinisikan semua kemungkinan state visual sebuah komponen. JavaScript hanya bertugas menambah atau menghapus class untuk beralih antar state. Tidak ada logika warna atau ukuran di JavaScript. 

/* CSS mendefinisikan SEMUA state */
.kartu-produk { border: 1px solid #ddd; transition: all 0.2s; }
.kartu-produk:hover { box-shadow: 0 4px 12px rgba(0,0,0,.1); }
.kartu-produk.dipilih { border-color: #2e7d32; background: #e8f5e9; }
.kartu-produk.stok-habis { opacity: 0.4; cursor: not-allowed; }

/* JavaScript hanya toggle class — tidak menyentuh warna sama sekali */
tombol.addEventListener('click', () => {
  kartu.classList.toggle('dipilih');
});
💡 Prinsip ini disebut State-Driven CSS dan adalah standar di semua framework modern (React className, Vue :class, Angular [ngClass]). Pelajari dari sekarang karena akan terus relevan.

JavaScript dari Perspektif CSS

JavaScript berinteraksi dengan CSS melalui tiga cara utama. Pertama, menambah/menghapus class via classList (paling direkomendasikan). Kedua, membaca nilai CSS yang dihitung via getComputedStyle. Ketiga, menulis inline style untuk nilai yang benar-benar dinamis via element.style.width = nilai + '%' — ini satu-satunya kasus di mana inline style dibenarkan.
