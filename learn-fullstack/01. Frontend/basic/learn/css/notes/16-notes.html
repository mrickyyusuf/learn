16 · Animations: Transitions, Keyframes, Transforms

Performa Animasi: GPU vs CPU

Sebelum membahas sintaks, pahami dulu prinsip performa ini karena sangat relevan untuk aplikasi retail dengan banyak animasi. Browser punya dua jalur rendering: CPU (lambat, menyebabkan jank) dan GPU (cepat, mulus). Properti yang hanya memicu Composite layer dieksekusi di GPU dan tidak memerlukan Layout atau Paint ulang.

Gunakan hanya dua properti untuk animasi yang smooth: transform (gerak, skala, rotasi) dan opacity (fade). Hindari menganimasikan width, height, top, left, margin, padding — semua ini memicu Layout ulang (mahal) pada setiap frame animasi.

Transitions — Animasi Antar State

Transition menganimasikan perubahan antara dua state CSS — biasanya dipicu oleh pseudo-class seperti :hover, :focus, atau perubahan class via JavaScript. Kamu tidak menulis "dari mana ke mana" — kamu hanya bilang "animasikan perubahannya".

Hover: warna
Hover: transform
Hover: shadow
Hover: all
/* Sintaks: property  duration  easing  delay */
.btn {
  transition: background-color 0.3s ease;      /* satu properti */
  transition: all 0.3s ease;                   /* semua properti */
  transition: transform 0.2s ease, opacity 0.3s ease; /* beberapa */
}

/* Easing functions yang paling umum */
transition-timing-function: ease;          /* lambat-cepat-lambat (default) */
transition-timing-function: ease-in;       /* mulai lambat */
transition-timing-function: ease-out;      /* akhiri lambat */
transition-timing-function: ease-in-out;   /* lambat di awal dan akhir */
transition-timing-function: linear;        /* kecepatan konstan */
transition-timing-function: cubic-bezier(0.25, 0.46, 0.45, 0.94); /* custom */

/* Praktik untuk kartu produk sembako */
.kartu-produk {
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}
.kartu-produk:hover {
  transform: translateY(-4px);
  box-shadow: 0 8px 24px rgba(0,0,0,0.15);
}

Transforms — Transformasi Elemen

Transform menggeser, memutar, menskalakan, atau memiringkan elemen tanpa mempengaruhi layout elemen lain. Ini adalah sifatnya yang paling penting — elemen lain tidak "tahu" bahwa transform sedang terjadi, sehingga tidak ada reflow. Transform juga adalah fondasi animasi performan karena dieksekusi di GPU. 

/* Translate — geser posisi */
transform: translateX(20px);         /* geser horizontal */
transform: translateY(-10px);        /* geser vertikal (naik) */
transform: translate(20px, -10px);   /* keduanya sekaligus */
transform: translateX(50%);          /* 50% dari LEBAR ELEMEN ITU SENDIRI */

/* Scale — ubah ukuran */
transform: scale(1.1);               /* 10% lebih besar */
transform: scale(0.9);               /* 10% lebih kecil */
transform: scaleX(1.5);              /* hanya horizontal */

/* Rotate — putar */
transform: rotate(45deg);            /* searah jarum jam */
transform: rotate(-90deg);           /* berlawanan */

/* Skew — miring */
transform: skewX(15deg);

/* Gabungan — urutan PENTING (dieksekusi dari kanan ke kiri) */
transform: translateY(-10px) scale(1.05) rotate(5deg);

/* transform-origin — titik pusat transformasi */
transform-origin: center;       /* default */
transform-origin: top left;     /* dari sudut kiri atas */
transform-origin: 50% 100%;     /* dari bawah tengah */

/* TRIK CENTERING dengan translate */
.tengah {
  position: absolute;
  top: 50%; left: 50%;
  transform: translate(-50%, -50%); /* geser balik setengah ukuran dirinya sendiri */
}

Keyframe Animations — Animasi Multi-Step

Berbeda dari transition yang hanya punya dua state (awal dan akhir), keyframe animation memungkinkan kamu mendefinisikan sequence animasi dengan banyak langkah dan berjalan secara otomatis tanpa interaksi pengguna.

Elemen ini fade in saat halaman dimuat
Stok Terbatas! 
/* Definisi keyframe */
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50%       { transform: scale(1.08); }
}

/* Skeleton loading shimmer — pattern yang dipakai Shopee, LinkedIn, dll. */
@keyframes shimmer {
  0%   { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}
.skeleton {
  background: linear-gradient(90deg, #e0e0e0 25%, #f5f5f5 50%, #e0e0e0 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
}

/* Terapkan animasi pada elemen */
.kartu-produk {
  animation-name: fadeIn;
  animation-duration: 0.5s;
  animation-timing-function: ease;
  animation-delay: 0s;
  animation-iteration-count: 1;         /* atau: infinite */
  animation-fill-mode: forwards;        /* pertahankan state akhir setelah selesai */
  /* Shorthand: nama duration easing delay iteration fill */
  animation: fadeIn 0.5s ease 0s 1 forwards;
}

/* Stagger effect: kartu ke-n muncul lebih lambat */
.kartu-produk:nth-child(1) { animation-delay: 0s; }
.kartu-produk:nth-child(2) { animation-delay: 0.1s; }
.kartu-produk:nth-child(3) { animation-delay: 0.2s; }
💡 Di aplikasi retail, animasi yang paling berguna adalah: skeleton loading (shimmer saat data dimuat), fade+slide in untuk list produk, pulse untuk badge urgency, dan spin untuk loading state. Semua menggunakan transform+opacity untuk performa GPU.
