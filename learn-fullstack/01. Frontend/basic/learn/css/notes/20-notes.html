20 · Best Practices: Performance & Accessibility

Performance: CSS yang Efisien

/* 1. GUNAKAN will-change untuk animasi kompleks */
/* Memberitahu browser untuk mengoptimasi elemen di GPU sebelumnya */
.kartu-animasi { will-change: transform; }
/* Tapi jangan berlebihan — setiap elemen dengan will-change mengkonsumsi memori GPU */

/* 2. PREFER transform dan opacity untuk animasi */
/* ✅ GPU-accelerated — tidak memicu Layout */
.smooth { transition: transform 0.2s, opacity 0.2s; }
/* ❌ Memicu Layout ulang setiap frame — "jank" pada animasi */
.jank { transition: width 0.2s, height 0.2s, top 0.2s; }

/* 3. CONTAIN — batasi efek rendering ke subtree */
.widget { contain: layout paint; } /* browser tidak perlu layout ulang seluruh halaman */
.kartu  { contain: content; }       /* shorthand untuk layout + paint + style */

/* 4. MINIMALKAN selector yang complex */
/* ❌ Browser harus cek SETIAP p di seluruh dokumen */
div > div > ul > li > p { color: red; }
/* ✅ Langsung ke target */
.list-item-text { color: red; }

/* 5. CSS Loading Strategy */
/* Critical CSS: inline di head untuk above-the-fold content */
/* Non-critical CSS: load async */
<link rel="preload" href="main.css" as="style" onload="this.rel='stylesheet'">

Accessibility: CSS yang Inklusif

/* 1. JANGAN PERNAH hapus focus indicator tanpa pengganti */
/* ❌ Ini membunuh aksesibilitas keyboard */
:focus { outline: none; }
/* ✅ Ganti dengan yang lebih estetik */
:focus-visible {
  outline: 3px solid #2e7d32;
  outline-offset: 3px;
  border-radius: 3px;
}

/* 2. Visually hidden — tersembunyi secara visual tapi TERBACA screen reader */
/* Berbeda dari display:none yang juga menyembunyikan dari screen reader */
.visually-hidden {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

/* 3. Respek preferensi animasi pengguna */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

/* 4. Kontras warna yang cukup */
/* WCAG AA: minimal 4.5:1 untuk teks normal, 3:1 untuk teks besar */
/* Alat cek: https://webaim.org/resources/contrastchecker/ */
/* ❌ Kontras rendah — sulit dibaca */
.text-bad { color: #9e9e9e; background: white; } /* rasio ~2.85:1 */
/* ✅ Kontras cukup */
.text-good { color: #616161; background: white; } /* rasio ~5.74:1 */

/* 5. Touch target minimal 44x44px untuk aksesibilitas mobile */
.btn-touch {
  min-height: 44px;
  min-width: 44px;
  padding: 0.75rem 1.25rem;
}

/* 6. Jangan gunakan warna saja untuk menyampaikan informasi */
/* ❌ Pengguna buta warna tidak bisa membedakan */
.valid   { color: green; }
.invalid { color: red; }
/* ✅ Tambahkan icon atau teks juga */
.valid::before   { content: "✓ "; }
.invalid::before { content: "✗ "; }

CSS Organization: Cara Menulis CSS yang Maintainable

/* URUTAN PENULISAN PROPERTI yang direkomendasikan */
.elemen {
  /* 1. Layout & Positioning */
  display: flex;
  position: relative;
  top: 0;
  z-index: 1;

  /* 2. Box Model */
  width: 100%;
  max-width: 400px;
  padding: 1rem;
  margin: 0 auto;
  border: 1px solid #ddd;
  border-radius: 8px;

  /* 3. Typography */
  font-family: Georgia, serif;
  font-size: 1rem;
  font-weight: 400;
  line-height: 1.6;
  color: #333;

  /* 4. Visual */
  background: white;
  box-shadow: 0 2px 8px rgba(0,0,0,.1);
  opacity: 1;

  /* 5. Animation */
  transition: transform 0.2s ease;
}
