17 · Responsiveness

Mobile-First vs Desktop-First

Ada dua pendekatan dalam menulis CSS responsif. Mobile-first: mulai dari style mobile sebagai default, lalu tambahkan media query min-width untuk layar lebih besar. Desktop-first: mulai dari desktop, tambahkan media query max-width untuk layar lebih kecil. Di era di mana mayoritas traffic e-commerce Indonesia datang dari mobile, mobile-first adalah standar yang lebih baik — kamu membangun untuk sebagian besar pengguna terlebih dahulu.

Media Queries

/* SINTAKS DASAR */
@media (min-width: 768px) {
  /* Style berlaku saat lebar viewport ≥ 768px */
  .product-grid { grid-template-columns: repeat(3, 1fr); }
}

/* MOBILE FIRST (direkomendasikan) */
/* Default: 1 kolom untuk mobile */
.product-grid { grid-template-columns: 1fr; }

@media (min-width: 640px) { /* tablet kecil → 2 kolom */
  .product-grid { grid-template-columns: repeat(2, 1fr); }
}
@media (min-width: 1024px) { /* desktop → 4 kolom */
  .product-grid { grid-template-columns: repeat(4, 1fr); }
}

/* MEDIA FEATURES lainnya */
@media (orientation: landscape) { .hero { height: 50vh; } }
@media (prefers-color-scheme: dark) { body { background: #121212; color: white; } }
@media (prefers-reduced-motion: reduce) {
  /* Matikan animasi untuk pengguna yang sensitif terhadap gerakan */
  *, *::before, *::after { animation: none !important; transition: none !important; }
}
@media print { .navbar, .footer { display: none; } }

/* COMMON BREAKPOINTS (gunakan yang sesuai konten, bukan device) */
/* 320px  — mobile kecil (jarang dipakai) */
/* 480px  — mobile besar */
/* 640px  — tablet kecil / landscape phone */
/* 768px  — tablet */
/* 1024px — desktop */
/* 1280px — desktop lebar */

Container Queries (CSS Modern)

Media query didasarkan pada ukuran viewport. Container query didasarkan pada ukuran container parent elemen itu sendiri. Ini jauh lebih berguna untuk komponen reusable seperti kartu produk — kartu yang sama bisa tampil 1 kolom atau 3 kolom tergantung di mana ia ditempatkan, bukan tergantung ukuran layar. 

/* Aktifkan container query pada parent */
.sidebar     { container-type: inline-size; container-name: sidebar; }
.main-content{ container-type: inline-size; container-name: main; }

/* Style berbeda tergantung lebar container */
@container (min-width: 400px) {
  .kartu-produk { display: flex; flex-direction: row; }
  .kartu-produk img { width: 120px; }
}

/* Kartu yang sama tampil berbeda di sidebar vs main content */
/* Dukungan browser: Chrome 105+, Firefox 110+, Safari 16+ */

Responsive Typography dengan clamp()

/* clamp(minimum, preferred, maximum) */
/* Font yang menyesuaikan ukuran layar secara fluid tanpa media query */

h1 { font-size: clamp(1.5rem, 5vw, 3rem); }
/* Di mobile (320px): 5vw = 16px < 1.5rem=24px → pakai 1.5rem */
/* Di tablet (768px): 5vw = 38.4px, antara 24-48px → pakai 38.4px */
/* Di desktop (1200px): 5vw = 60px > 3rem=48px → pakai 3rem */

h2 { font-size: clamp(1.25rem, 3vw, 2rem); }
p  { font-size: clamp(0.9rem, 1.5vw, 1.1rem); }

/* Responsive spacing */
.section { padding: clamp(2rem, 5vw, 4rem); }
.gap     { gap: clamp(0.75rem, 2vw, 1.5rem); }
💡 Trio responsif modern tanpa media query: clamp() untuk typography, repeat(auto-fill, minmax()) untuk Grid, dan flex-wrap: wrap untuk Flexbox. Dengan ketiganya, banyak layout responsif bisa dibuat tanpa menulis satu media query pun.
